# 投资人与餐饮客户 20 分钟演示脚本

**场景**：中餐正餐，云端数据，面向投资人与餐饮客户展示业务逻辑

**数据**：30 条订单，33 道菜品，7 大分类，全部使用真实菜单菜品；会员、优惠、渠道、厨房工单、现金、结算、自助点餐等完整业务数据

---

## 演示前准备

1. **种子数据**：执行 `cd pos-backend && node scripts/seed.js` 写入演示数据
2. **登录**：`testadmin@restro.local` / `12345678` 或 `admin@restro.local` / `Admin@12345`
3. **语言**：演示建议使用中文界面（右上角切换）

---

## 时间轴（约 20 分钟）

| 分钟 | 模块 | 演示要点 |
|------|------|----------|
| 0–2 | 登录与首页 | 登录后进入首页，展示营业额、订单数、会员数、菜品数等指标 |
| 2–4 | 桌台管理 | 展示 12 张桌台，中餐客户名（张明、李芳等），桌台状态（已订/空闲） |
| 4–7 | 菜单与点餐 | 7 大分类（凉菜/热菜/汤羹/主食/饮品/甜品/酒水），33 道菜品，¥ 定价；选菜品→加入购物车→下单 |
| 7–10 | 订单中心 | 30 条订单，状态流转（进行中→待出餐→已完成/已取消），支付方式（现金/在线） |
| 10–13 | 厨房生产/KDS | 工位路由（凉菜→COLD、热菜→HOT_LINE、酒水→BAR、甜品→DESSERT），工单状态 |
| 13–15 | 会员与优惠 | 会员等级（金/银/铜/铂金）、积分、储值；优惠规则（新客9折、会员满减）、优惠券 |
| 15–17 | 渠道与设备 | 饿了么/美团/堂食渠道连接；设备（KDS、打印机）在线状态 |
| 17–18 | 现金与结算 | 开班/交班、现金流水；结算批次、核心财务指标 |
| 18–20 | 自助点餐 / 合作方 API | 桌码扫码下单；Partner API Key 对接第三方系统 |

---

## 关键话术

- **业务闭环**：从顾客到店/扫码→点餐→厨房制作→出餐→支付→结算，全流程打通
- **真实数据**：订单、菜品、会员、支付均为真实结构，可对接生产环境
- **多门店/多区域**：支持总部、区域、门店三级组织
- **合规与安全**：审计日志、PII 脱敏、审批工作流

---

## 技术说明

- 后端菜单与订单统一使用 `MenuCatalogItem`，订单行 `name` 必须与菜品 `name` 完全一致
- 厨房工位由 `kitchenRouting.js` 按菜品名称关键词自动路由
- 货币：演示数据为 CNY（¥），可配置为其他币种
